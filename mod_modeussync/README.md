# Задания из Modeus (`mod_modeussync`)

Activity-плагин предоставляет преподавателю ручной этап создания оцениваемых элементов по заданиям, уже полученным `tool_modeussync` от SyncService.

## Установка и зависимость

Плагин требует установленный `tool_modeussync` совместимой версии. Он использует очередь, настройки подключения (`syncservice_base_url`, `internal_api_key`) и публичный клиент `tool_modeussync\service\SyncService`; контракты SyncService не изменяются.

На курсе может существовать ровно один экземпляр `mod_modeussync`. Он создаётся системой только при наличии заданий в очереди. Добавление и удаление через стандартный интерфейс Moodle запрещены; отсутствующий экземпляр восстанавливает фоновая задача.

## Работа преподавателя

1. Откройте «Задания из Modeus» на странице курса.
2. Для каждой строки выберите тип: «Задание» (`assign`, выбран по умолчанию) или «Тест» (`quiz`).
3. Нажмите «Создать».

Название, `course_modules.idnumber` и максимальная оценка каждого созданного элемента берутся из `courseData`. Moodle хранит максимум `assign` как целое число, поэтому дробное значение для выбранного `assign` отклоняется без округления; его можно создать как `quiz`, который поддерживает дробной максимум. `quiz` создаётся пустым, с одной попыткой и методом итоговой оценки «Лучшая оценка»; вопросы преподаватель добавляет самостоятельно.

При частичной ошибке уже созданные элементы сохраняются, а повторная попытка обрабатывает только отсутствующие. `POST /sync` вызывается только после создания или обнаружения всех элементов. Если ошибка произошла только при `/sync`, повторная попытка не создаёт элементы заново и повторяет только отправку неизменной связки `id_modeus`/`id_lms`.

Созданные `assign` и `quiz` затем передаются через штатную задачу `tool_modeussync\task\push_courses`; технический `modeussync` не экспортируется.

## Права

- `mod/modeussync:view` — просмотр очереди;
- `mod/modeussync:manage` — выбор типов, создание элементов и повторная отправка;
- `mod/modeussync:addinstance` — служебная capability без стандартных назначений ролям; сама по себе не разрешает ручное добавление.
