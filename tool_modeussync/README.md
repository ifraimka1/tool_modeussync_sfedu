## Плагин для интеграции Modeus с Moodle.

### Фоновые задачи

Фоновых задач (анг. job) четыре:

- pull_courses (создание курсов Moodle по РМУП Modeus)
- push_courses (передача курсов в Modeus)
- pull_members (зачисление пользователей на курсы Moodle по составу команд РМУП Modeus)
- push_grades (передача оценок в Modeus для проставления в Успеваемость)

Задачи по-умолчанию запускаются каждые 5 минут (можно изменить в настройках).

### Особенности работы задач

#### pull_courses

- В БД Moodle у курсов и их модулей, которые были созданы по данным РМУПа (опция "Создать курс"), в поле idnumber проставляется уникальный идентификатор. Этот идентификатор опционален с точки зрения синхронизации и после завершения привязки к РМУПу может быть удален.
- В Moodle у курсов есть уникальное "короткое имя" или "shortname". В курсах, которые были созданы по данным РМУПа (опция "Создать курс"), в это поле прописывается идентификатор РМУПа. С точки зрения синхронизации этот идентификатор не важен и может быть изменен на другой.

#### pull_members

- Плагин отчисляет с курсов пользователей, не найденных в связанном РМУПе. Отчисление можно отключить отдельно для студентов и преподавателей в настройках плагина (параметры `unenrol_students` и `unenrol_teachers`)

#### push_courses

- Для отбора курсов для отправки в Modeus плагин использует ограничение по их "возрасту". По умолчанию плагин отправляет курсы не старше 1 года (более старые считаются нерелевантными). Ограничение можно изменить в настройках плагина (параметр `courses_max_interval_minutes`).
- Для мониторинга изменений в модулях курсов используется данные из стандартного журнала событий Moodle, соответственно, он должен быть включен (Администрирование — Плагины — Стандартный журнал событий).

#### push_grades

- Для отбора оценок для отправки в Modeus плагин использует ограничение по их "возрасту". По умолчанию отправляются оценки не старше 4 недель. Ограничение можно изменить в настройках плагина (параметр `grades_max_interval_minutes`)
